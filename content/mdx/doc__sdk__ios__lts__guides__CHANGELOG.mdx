## \[2.13.0-LTS\] - 2025-03-07

### Changed

-   The property `groupsService` from `ServicesManager` has been removed. Use property `groupsManager` from `ContactsManagerService` instead.
-   The property `locationService` from `ServicesManager` has been removed because the location is used only in `ConversationsManagerService`.
-   The property `urlMetadataService` from `ServicesManager` has been removed. Use property `urlMetadataManager` from `ConversationsManagerService` instead.
-   The following `ConferencesManagerService` methods has been removed from the public header :
    -   `-(void) rejectIncomingConference:completionHandler:`
    -   `-(void) join:(NSString *)roomId completionHandler:(ConferenceManagerErrorBlock) completionHandler`
-   The following methods from `CallsManager` have now a completionBlock:
    -   `-(BOOL) hold:(Call* _Nonnull) activeCall`
    -   `-(BOOL) retrieve:(Call* _Nonnull) heldCall`
    -   `-(BOOL) swapCurrentCalls`
    -   `-(void) setMutedState:(BOOL) muted forCall:(Call* _Nonnull) mCall`
-   The following methods from `TelephonyService` have now a completionBlock:
    -   `-(void) swapCall:(Call* _Nonnull) activeCall withHeldCall:(Call* _Nonnull) heldCall`
    -   `-(void) blindTransferCall:(Call* _Nonnull) activeCall toPhoneNumber:(PhoneNumber* _Nonnull) phNumber`
-   The following deprecated methods (without completionHandler) have been removed:
    -   `-(void) deleteGroup:(Group *_Nonnull) group`
    -   `-(void) updateGroup:(Group *_Nonnull) group withNewComment:(NSString *_Nonnull) newComment`
    -   `-(void) addContact:(RainbowContact *_Nonnull) contact inGroup:(Group *_Nonnull) group`
    -   `-(void) removeContact:(RainbowContact *_Nonnull) contact fromGroup:(Group *_Nonnull) group`
-   The method `-(void) fetchRoomWithOpenInviteId:withCompletionHandler:` has been replaced by `-(void) joinBubbleWithPublicId:completionHandler:`. The completionHandler has now a new string `lobbyRoomId` in case the lobby is enabled and the user is in, and not automatically added to the bubble.
-   The method `-(void) validateRoomOpenInviteId:withCompletionHandler:` has been replaced by `-(void) fetchRoomPublicData:completionHandler:`. The completionHandler has now a room with id and name, but also a boolean `isConferenceStarted`.
-   The properties `isVideoHeld`, `isSharingHeld` and the method `isMediaHeldForSharing` from a `RoomConfParticipant` has been removed and replaced by `isMediaHeld`. The new property `isSharingType` can be used to know if the participant is a sharing type or video.

    -   The method `isLeaderInVoiceGroup` has been renamed to `isManagerInOneMAGroup`.
    -   The method `getFirstLeaderAssistantGroup` has been renamed to `getFirstManagerAssistantGroup`.
    -   The method `getLeaderAssistantGroups` has been renamed to `getManagerAssistantGroups`.
    -   The method `isActiveInAllLeaderAssistantGroups` has been renamed to `isActiveInAllManagerAssistantGroups`.
    -   The method `changeMyStatusInAllLeaderAssistantGroupsToActive` has been renamed to `changeMyStatusInAllManagerAssistantGroupsToActive`.
-   The class `VoiceGroupMember` has been renamed to `TelGroupMember`.
    -   The enum `VoiceGroupMemberStatus` has been renamed to `TelGroupMemberStatus`.
    -   The enum `VoiceGroupMemberRole` has been renamed to `TelGroupMemberRole`.
    -   The enum `VoiceGroupMemberConnectedStatus` has been renamed to `TelGroupMemberConnectedStatus`.
    -   The method `isLeader` has been renamed to `isManager`.
    -   The method `isManager` has been renamed to `isAdmin`.
    -   The method `isManagerOnly` has been renamed to `isAdminOnly`.
    -   The method `isManagerAndAgent` has been renamed to `isAdminAndAgent`.
-   The method `getSyncAvatarInCacheForContact` has been removed from `ContactsManagerService` public header.
-   The class `ContactPatternSearchCriteria` has been renamed `ContactsPatternSearchCriteria`.
-   The method `simulateDidReceiveMessage` has been removed fromm `ConversationsManagerService` public header.
-   The methods `createPollInRoom` and `updatePoll` from `PollsManager` have now a new parameter `modifyVote`.
-   The method `getLeaderAssistantStateString` from SipwiseTelephonySce has been removed (UI choice).

## \[2.12.0-LTS\] - 2024-07-23

### Changed

-   The following internal methods have been removed. You should use `updateDisplayedVideos` instead, to automatically hold / unhold the displayed videos.
    -   `holdRemoteVideo`
    -   `unholdRemoteVideo`
    -   `holdActiveSpeakerVideo`
    -   `unholdActiveSpeakerVideo`

### \[SDK for iOS 2.4.1 LTS\] - June 2022

***API Changes***

-   Since version 2.4.0 LTS, the pushes (IM and VoIP) are automatically encrypted by the server. Therefore, the iOS app needs to decrypt the received IM push by calling method `decryptNotificationContentWithUserInfo` from notificationsManager (the VoIP push is directly decrypted in the SDK):

In the notification service extension :

```
NSString *crypt = [userInfo valueForKeyPath:@"aps.crypt"];
if (crypt != nil) {
    NSDictionary *decryptedUserInfo = [[ServicesManager sharedInstance].notificationsManager decryptNotificationContentWithUserInfo:request.content.userInfo];
} else {
    // Push is not encrypted
}
```

***Other Changes***

-   The documentation to update or delete avatar has been updated
-   The documentation to update user phone numbers has been updated

### \[SDK for iOS 2.4.0 LTS\] - April 2022

***API Changes***

It is accessible by `[ServicesManager sharedInstance].myUser.customData`. It is updatible by `[[ServicesManager sharedInstance].contactsManagerService updateUserWithFields:@{@"customData" : yourCustomDataDictionary} withCompletionBlock:nil];`

***Other Changes***

-   Starting from Rainbow SDK for iOS version 2.4.0 LTS, the IM messages are encrypted using CryptoSwift. We have also started to migrate our API calls to use Alamofire library. Therefor, if not already done, you need to add the dependencies in your application in order to build with this version.

Easy way using Swift Package Manager (but using Carthage is also working):

```
.package(url: "https://github.com/krzyzanowskim/CryptoSwift.git", .upToNextMajor(from: "1.4.3"))
.package(url: "https://github.com/Alamofire/Alamofire.git", .upToNextMajor(from: "5.5.0"))
```

### \[SDK for iOS 1.67.0\] - February 2020

***API/Methods Changes***

##### Mark as read all new received messages

The method to mark as read all new messages received for a conversation in `ConversationsManagerService` has changed.

Internally: - the old method sends an XMPP message for each unread messages to mark as read - the new method sends a REST request to mark all unread messages as read

The name of the method has also changed: - Old name: `-(void) markAsReadByMeAllMessageForConversation:(Conversation *) conversation` - New name: `-(void) markAllMessagesAsReadForConversation:(Conversation *) conversation`

##### Mark as read any new message received

The method to mark as read any new message received for a graphically opened conversation has changed.

The method `-(void) markAsReadByMeAllMessageForConversation:(Conversation *) conversation` has been removed. It was called on UI side any time we receive a new message in the current conversation.

The new way is to set the property `automaticallySendMarkAsReadNewMessage` from a `Conversation` object at : - YES for example in viewWillAppear in the conversation view - NO for example in viewWillDisappear in the conversation view

The SDK will then manage the mark as read behavior.

##### Others changes

-   The method `createAutoAcceptedRoom` has been added to create a bubble without invitation for any new user added. See guide **Managing bubble** to get more information.
