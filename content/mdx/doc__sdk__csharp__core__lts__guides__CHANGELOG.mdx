#### Info

-   New `Sms` service permits to send SMS. Specific rights must be defined to use this new service.
-   Add resilience at HTTP request level:
    -   If necessary up to 4 more attempts are automatically made by the SDK (after 200, 400, 800 and 1600 ms or using **RetryAfter** header value returned by the server).
    -   It's possible to define your own logic (for tests, depending on the domain used, etc.) by defining your own `Http.DelegatingHandler` - see `HttpRetryMessageHandler` object for more details.
-   It's now possible to join several conferences in same time using the same account - need to set `JoinMultipleConferences` to true in `Restrictions` object

-   `Application` service:
    -   Avoid crash if xmpp or auto-reconnection object has been been already closed / disposed.
-   `Bubbles` service:
    -   Request information from server about an unknown `Bubble` when a message about user management is received (can occurred when guests are used).
    -   Avoid crash when guests are used.
    -   Avoid to request too often conference snapshot.
    -   `BubbleLink` can be **public** (owner and moderators can acces to it) when created by the bubble's owner or **private** when created by a moderator (only visible to him).
-   `Conversations` service:
    -   Check Peer's Id validity before to request server to create a conversation.
    -   When adding a `Conversation` to the cache, ensure to know the associated `Peer` (as `Contact` or `Bubble`).
-   `InstantMessaging` service:
    -   Use cache first to get a `Conversation` instead to always create it.
-   `MediaPublication` object:
    -   Ensure to set empty string for **MediaId, MediaName, MediaDescription, PeerResource** instead of null.

#### Added

-   Log Http Headers set specifically using the SDK
-   `Administration` service:
    -   `UpdateApplicationAsync()` method
-   `Application` service:
    -   `GetIniFullPath()` method
-   `Bubbles` service:
    -   `GetBubbleLinkAsync()`: optional parameter **asPublic** (true by default) to get **public** (created by the owner) or **private** (created by a moderator) Bubble Link
-   `HttpClient` object:
    -   `SetDelegatingHandler()`, `DefaultDelegatingHandler()` methods
-   `HttpRetryMessageHandler` object:
    -   used as default `Http.DelegatingHandler` by the `HttpClient`
-   `Peer` object:
    -   `CanBeUsedInConversation()`, `HasValidId()`, `HasValidJid()`, `HasValidIdentifier()` static methods

### \[3.0.0-sts.2\] - 2024-07-17

#### Info

-   Add notion of **stackable** and **unstackable** events - both to avoid to trigger several times same event in short period.

### \[3.0.0-sts.1\] - 2024-07-01

#### Info

-   Contains nearly all features of v2.X
-   New features will only been added in V3.X
-   Simplification is the goal of this new release:
    -   Use Async/Await pattern
    -   No more needs to wait `InitializationPerformed`. Waiting the end of the login is enough.
    -   `Peer` object is used centrally to describe a `Bot`, `Bubble`, `CallParticipant`, `Channel`, `ConferenceParticipant`, `Contact`, `Conversation`, `MediaService` (WebRTC context), `DynamicFeed` (WebRTC context) or a user known only by a phone number.
    -   Implicit operator permits to create `Peer` object from `Bot`, `Bubble`, `CallParticipant`, `Channel`, `ConferenceParticipant`, `Contact`, `Conversation`
    -   Avoid usage of id/jid in methods and used objects instead
-   Samples will be updated in consequence

#### Removed

-   `AckMessage` features in `InstantMessaging` service. Use instead `ApplicationMessage` - more versatile and simple
-   `AdHocCommand` features in `InstantMessaging` service. Use instead `ApplicationMessage` - more versatile and simple
-   `InitializationPerformed` event from `Application` service. `ConnectionStateChanged` event is enough.

#### Added

-   `Restrictions` object: `AcceptBubbleInvitation` (False by default). If set to True, automatically accept pending bubble invitation.
-   `Restrictions` object: `AcceptUserInvitation` (False by default). If set to True, automatically accept pending user invitation.
