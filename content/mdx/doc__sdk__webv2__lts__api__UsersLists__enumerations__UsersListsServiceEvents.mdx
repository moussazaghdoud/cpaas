# Enumeration: UsersListsServiceEvents

UsersLists service events send to other services Can listen to it via `UsersListsService.subscribe()` API

## Events

### ON\_USERS\_LIST\_CREATED

> **ON\_USERS\_LIST\_CREATED**: `"ON_USERS_LIST_CREATED"`

This `RBEvent` is sent when a new users list has been created.

#### Event Data

data - The new created users list (as UsersList model)

---

### ON\_USERS\_LIST\_DELETED

> **ON\_USERS\_LIST\_DELETED**: `"ON_USERS_LIST_DELETED"`

This `RBEvent` is sent when a users list has been deleted.

#### Event Data

data - The id (as string) of the users list that has been deleted.

---

### ON\_USERS\_LIST\_PROFILE\_UPDATED

> **ON\_USERS\_LIST\_PROFILE\_UPDATED**: `"ON_USERS_LIST_PROFILE_UPDATED"`

This `RBEvent` is sent when the list information (name, description or favorite status) has been modified.

#### Event Data

data - The updated users list (as UsersList object)

---

### ON\_USER\_ADDED\_TO\_LIST

> **ON\_USER\_ADDED\_TO\_LIST**: `"ON_USER_ADDED_TO_LIST"`

This `RBEvent` is sent when a user has been added to the list.

#### Event Data

user - The user (as User object) added to the list

#### Event Data

list - The users list (as UsersList object) to which the user has been added.

---

### ON\_USER\_REMOVED\_FROM\_LIST

> **ON\_USER\_REMOVED\_FROM\_LIST**: `"ON_USER_REMOVED_FROM_LIST"`

This `RBEvent` is sent when a user has been removed from the list.

#### Event Data

userId - The id (as string) of the user removed from the list.

#### Event Data

list - The users list (as UsersList object) to which the user has been added.
