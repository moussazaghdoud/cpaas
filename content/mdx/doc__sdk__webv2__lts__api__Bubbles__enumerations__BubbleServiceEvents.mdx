# Enumeration: BubbleServiceEvents

Bubble service events send to other services Can listen to it via `bubbleService.subscribe()` API

## Events

### ON\_INVITATION\_RECEIVED

> **ON\_INVITATION\_RECEIVED**: `"ON_INVITATION_RECEIVED"`

This RBEvent is send when there's a new invitation to join bubble that is received

#### Param

the bubble related

---

### ON\_MY\_USER\_PRIVILEGE\_UPDATE

> **ON\_MY\_USER\_PRIVILEGE\_UPDATE**: `"ON_MY_USER_PRIVILEGE_UPDATE"`

This `RBEvent` is send when a there's an update on my user privilege (owner, moderator, user)

#### Param

the bubble related

#### Param

the user privilege

#### Param

the user status

---

### ON\_MY\_USER\_STATUS\_UPDATE

> **ON\_MY\_USER\_STATUS\_UPDATE**: `"ON_MY_USER_STATUS_UPDATE"`

This `RBEvent` is send when a there's an update on my user status (invited, accepted, deleted,...)

#### Param

the bubble related

#### Param

the user status

---

### ON\_NEW\_BUBBLE\_CREATED

> **ON\_NEW\_BUBBLE\_CREATED**: `"ON_NEW_BUBBLE_CREATED"`

This RBEvent is send when there's a new bubble is created

#### Param

the new bubble related

#### Param

if TRUE, that means that the new bubble is created but some/all of the invited users are not successfully added; Only for bubbles/rooms created by the user

#### Param

The DbId of the contacts that could not be invited to the bubble

---

### ON\_SERVICE\_STARTED

> **ON\_SERVICE\_STARTED**: `"ON_SERVICE_STARTED"`

This `RBEvent` is send when the bubble service is started and ready to be used (also fired after loss of network and reconnection)

---

### ON\_USER\_STATUS\_UPDATE

> **ON\_USER\_STATUS\_UPDATE**: `"ON_USER_STATUS_UPDATE"`

This `RBEvent` is send when a there's an update on the bubbles participants status (invited, accepted, etc)

#### Param

the bubble related

#### Param

the user inside the bubble related

#### Param

the user status
