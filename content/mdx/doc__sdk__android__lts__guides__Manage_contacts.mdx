## Manage contacts

---

Welcome to the tutorial on managing contacts using **Rainbow SDK for Android !** By the end of this guide, you'll be able to integrate contact management features into your Android application with ease. We'll cover accessing the user's roster, creating and managing personal contacts, and listening to changes in this information for updating your application's UI.

**Rainbow concept**

**Description**

**IRainbowContact**

Represents the public interface for accessing information about Rainbow contacts, including the logged-in user. This interface is utilized and returned by various SDK functions.

**Roster**

In XMPP, one's contact list is called a roster, which consists of any number of specific roster items, each roster item being identified by a unique JID (usually of the form contact@domain). For more information, refer to [RFC 6121](https://xmpp.org/rfcs/rfc6121.html#intro-requirements).

**Personal contacts**

A personal contact is one that you create and manage, exclusive to your account. You have full control over these contacts, allowing you to customize and utilize them according to your preferences

### Access the user's roster

The user's roster contains contacts that the user has invited, serving as both a contact list and a means of accessing the presence of contacts.

Key Points to Consider:

1.  **Roster Compositions:** The roster consists of contacts explicitly invited by the user. These contacts can be individuals or entities with whom the user interacts frequently.
2.  **Invitation Mechanism:** Users can send invitations by specifying the contact's email address. Upon sending, a visibility request is initiated. If the recipient accepts the request, they are added to the user's roster.

To access the roster, you can use the `roster` property of the `Contacts` module. This property returns an [ArrayItemList](/docs/sdk/android) containing objects of type `IRainbowContact`. You can then utilize this interface to display the required information in your application. Below are code examples demonstrating how to implement roster functionality in both Kotlin and Java.

-   [kotlin](#codegroupTab-roster_kotlin)
-   [java](#codegroupTab-roster_java)

### Retrieve a contact

The contacts module exposes several methods for retrieving an instance of `IRainbowContact`. One common method is to find a contact by its ID. This is particularly useful for scenarios such as displaying a contact's details when a RecyclerView item is clicked, opening a screen containing the contact's information.

If the user's information is not present in the cache, the `getContactById()` method will return null. In such cases, it's advisable to retrieve the information directly from the server. However, it's important to note that depending on the visibility settings of the searched user, the retrieved information may vary for privacy reasons.

-   [kotlin](#codegroupTab-get_by_id_kotlin)
-   [java](#codegroupTab-get_by_id_java)

```kotlin
class ContactDetailsFragment : Fragment() {

    private var contact: IRainbowContact? = null

    override fun onCreateView(inflater: LayoutInflater, container: ViewGroup?, savedInstanceState: Bundle?): View {
        arguments?.let {
            val contactID = it.getString("ContactID") ?: error("Contact ID is mandatory")
            contact = RainbowSdk.instance().contacts().getContactFromId(contactID)

            if (contact == null) {
                fetchContactInformation(contactID)
            } else {
                updateLayout()
            }

        } ?: error("This fragment cannot be shown without bundle")

        // Inflate the layout for this fragment
        return inflater.inflate(R.layout.fragment_contact_details, container, false)
    }


    private fun fetchContactInformation(contactID: String) {
        RainbowSdk().contacts().fetchContactDataById(contactID, object : RainbowListener<IRainbowContact, UserRepository.FetchUserDataError> {
            override fun onSuccess(data: IRainbowContact) {
                contact = data
                updateLayout()
            }

            override fun onError(error: RainbowError<UserRepository.FetchUserDataError>) {
                // Handle error cases
                if (error.errorType == FetchUserDataError.CONTACT_TERMINATED) {
                    // Contact has deleted their account
                }
            }
        })
    }

    private fun updateLayout() {
        // Update views
    }
}
```

### Listen to contact changes

You can listen to changes on an object of type `IRainbowContact`, allowing your application to be notified when the contact in question undergoes updates or changes its presence state.

*When registering a listener at some point in your application's lifecycle, it's crucial to remove it before reaching the end of the lifecycle to prevent potential memory leaks.*

-   [kotlin](#codegroupTab-contact_update_kotlin)
-   [java](#codegroupTab-contact_update_java)

```kotlin
class ContactDetailsFragment : Fragment() {

    private var contact: IRainbowContact? = null
    private val contactListener = object : IRainbowContact.IContactListener {
        override fun contactUpdated(updatedContact: IRainbowContact) {
            // Handle update of the contact
        }

        override fun onPresenceChanged(contact: IRainbowContact, presence: RainbowPresence?) {
            // Handle presence change of the contact
        }

        override fun onCompanyChanged(companyId: String?) {
            // Optional callback to be notified when the company associated with the contact changes.
            // This can be useful for updating UI elements or performing additional actions when the contact's company information is updated.
        }
    }

    override fun onCreateView(inflater: LayoutInflater, container: ViewGroup?, savedInstanceState: Bundle?): View {
        // Inflate the layout for this fragment
        return inflater.inflate(R.layout.fragment_contact_details, container, false)
    }


    override fun onResume() {
        super.onResume()
        contact?.registerChangeListener(contactListener)
    }

    override fun onPause() {
        super.onPause()
        contact?.unregisterChangeListener(contactListener)
    }
}
```
