# Documentation

Glossary

## Glossary

---

The list below defines some of the basic terminology of the Rainbow for Developers and Rainbow.  
  

If you are new to Rainbow or have any question regarding Rainbow usage, you can go to the [Rainbow Help Center](https://support.openrainbow.com/hc/en-us).

#### Admin

This represents an administrative role in Rainbow. Users having an admin role can manage a company (for example company\_admin role) or an organization ( E.g. org\_admin)

#### Availability

The availability represents all the information you share with your contacts regarding how they can reach you. The availability contains your application and phone presence and additionally your calendar presence information.  
  

#### Bubble

A bubble is a workspace designed for real time collaboration between Rainbow users. Bubble members can be organizer or member. They can chat, share files, start audio or video conferences, share their screens,... You can find more information about Rainbow bubbles usage on the Rainbow Help Center: [Bubbles](https://support.openrainbow.com/hc/en-us/sections/360003136859-Bubbles).  
  

#### Call

A call is an audio only or audio and video interaction between two or more users. A call can be a CTI (Computer Telephony Integration) based call, a webRTC based call or a mixed CTI and webRTC call. A call can contain an additional application or screen sharing leg. A call always takes place into a conversation.  
  

#### Channel

A channel is a Rainbow feature which represents a mass notification medium for sending information to a large number of users (several thousands in one time). More about Rainbow Channels on the Rainbow Help Center: [Channels](https://support.openrainbow.com/hc/en-us/sections/360003495700-Channels).  
  

#### Company

Each user in Rainbow belongs to a company. When creating your Rainbow account, you belongs to the default Rainbow company. Then you can join an existing company to collaborate with your colleagues or create your own company. More information about Rainbow Companies on the Rainbow Help Center: [What is a Rainbow Company?](https://support.openrainbow.com/hc/en-us/articles/115001371964)  
  

#### Contacts

Contacts are Rainbow users that are in your network. More information here: [How to add someone to my contacts](https://support.openrainbow.com/hc/en-us/articles/115000214450-How-to-Add-Someone-to-My-Contacts-#web)  
  

#### Conversation

A conversation represents the history of exchanges between you and another Rainbow user. This is all the interactions you had (i.e. calls, instant messages, files shared) and that you can retreive. A conversion is persistent. At any time, you could retrieve older interactions you did with someone.  
  

#### Deployed application

A deployed application is a third party application that has been granted to connect to the Rainbow production platform.  
  

#### Developer

This is a specific role a Rainbow user gains when registering to the Rainbow for Developers portal. This role is needed to create and manage third party applications.  
  

#### Developer Sandbox account

This is a specific Rainbow account you have to use to connect to the Rainbow Developers sandbox platform for managing your Sandbox test accounts. Developer sandbox account credentials are different from the main Rainbow account credentials used to connect to the production platform.  
  

#### Instant messaging

Instant Messaging is a Rainbow feature which allows to send and receive messages based on text, emoji, files and images.  
  

#### File sharing

The file sharing is a Rainbow feature allowing users to share files. More information can be found here [How to manage my Rainbow sharing](https://support.openrainbow.com/hc/en-us/articles/360000134050-How-to-Manage-My-Rainbow-Sharing-)  
  

#### Group

Contacts in Rainbow can be gathered by defining groups. Groups allow to categorize and organize your list of contacts.  
  

#### Network

This is the list of contacts with whom you are sharing mutually your availability: They see your availability, you see their availability.  
  

#### Organization

An organization is a set of companies where users can see each other and collaborate.  
  

#### Presence

The presence is a status in Rainbow that helps your contacts understanding if you can or not be reached at a time. More information can be found here: [What does my contacts status mean](https://support.openrainbow.com/hc/en-us/articles/115000046710-What-Does-My-Contacts-Status-Mean-).  
  

#### Privacy

The privacy is a security mechanism that exists in Rainbow to protect your personal information from being accessible by other Rainbow users and to avoid unexpected interaction with Rainbow users that are out of your circle of activity (company or organization).  
  

#### Production platform

This is the Rainbow platform where you are connected when using the AL-Enterprise Rainbow applications.  
  

#### Profile

A profile (or a subscription) is a list of features a user can subscribe for. Several profiles exist in Rainbow: Essential, Business, Enterprise. By default, a user has the Essential profile. Third-party applications (CPaaS) overide the user profile to offer more.  
  

#### Push notifications

The push notifications is the possibility for your mobile application to receive notifications from Rainbow when in background.  
  

#### Rainbow guest

Represents a guest user in Rainbow. This is a restricted role a user can have (often associated to anonymous user) dedicated to third-party applications (CPaaS) and with a limited duration of life. Note: A Rainbow guest can't connect to a Rainbow application provided by AL-Enterprise.  
  

#### Rainbow user

Represents a user in Rainbow. This is the default role you have when joining Rainbow. A Rainbow user can access to all the collaborative services of Rainbow depending their profile.  
  

#### REST API

The REST API are the Rainbow API we published and documented that you can use for accessing the Rainbow services. More information in the [REST APIs](/apis) documentation.  
  

#### Role

The role is the right you have in Rainbow. Several rights exist in Rainbow: guest, user, admin, developer, etc... A Rainbow user has the user role by default. A user may have more than one role (for example user, developer and admin).  
  

#### Sandbox platform

The Sandbox platform is a dedicated Rainbow platform for developers. The sandbox platform is not connected to the production platform and use different accounts. Called the Rainbow Sandbox platform for developers.  
  

#### Sandbox test accounts

Sandbox test accounts are specific accounts that you have to use for connecting and testing your application on the Rainbow Sandbox platform for developers. You can create Sandbox test accounts using Rainbow for Developers **Sandbox** section or the Rainbow CLI tool.  
  

#### SDK

A SDK is a library and a set of tools, samples and starter-kits provided by ALE International that allows the creation of applications for Rainbow.  
  

#### System

A system represents a PABX (AL-Enterprise OmniPCX Office or OmniPCX Enterprise or third party) that can be connected to Rainbow. A system is attached to a company.  
  

#### Third party application

This is a Rainbow compliant application or plugin created from the Rainbow for Developers portal.  
  

#### vCard

The vCard is my personal information. These information are shared with my contacts. Due to the privacy, Rainbow users who are not in my network have access to only a limited subset of my personal information.  
  

#### Visibility

The visibility in Rainbow is the way others users can found and interact with you. If you have a `private` visibility only users from the same company or whom you share the network will be able to interact with you. `public` means that every users can interact with you. More information can be found in guide [Visibility and privacy rules](/docs/hub/users-visibility-and-privacy).
