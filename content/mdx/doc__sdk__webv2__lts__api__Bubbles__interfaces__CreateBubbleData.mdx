# Interface: CreateBubbleData

The `CreateBubbleData` interface describes the data that can be send to the create bubble API

## Example

```ts
const data = {
     "topic": "my bubble topic",
     "createConversation": true,
     "autoAcceptInvitation": true
}
createBubble("my bubble name", data);
```

## Properties

### autoAcceptInvitation?

> `optional` **autoAcceptInvitation**: `boolean`

When set to true, allows to automatically add participants in the bubble (default behavior is that participants need to accept the bubble invitation first before being a member of this bubble) `Default : FALSE`

---

### bubbleAvatar?

> `optional` **bubbleAvatar**: `any`

Avatar image for the bubble

---

### contacts?

> `optional` **contacts**: `Contact`\[\]

List of contacts that we want to invite to the bubble

---

### disableTimeStats?

> `optional` **disableTimeStats**: `boolean`

If TRUE, in case of conference in this bubble the talking time stats will be disabled `Default : FALSE`

---

### guestEmails?

> `optional` **guestEmails**: `string`\[\]

List of emails that we want to invite to the bubble (will receive an invitation email)

---

### muteUponEntry?

> `optional` **muteUponEntry**: `boolean`

In case of conference in this bubble, all participants will be muted upon join `Default : FALSE`

---

### noConversation?

> `optional` **noConversation**: `boolean`

If TRUE, the server will NOT create a conversation for this bubble `Default : FALSE`

---

### noEntryTone?

> `optional` **noEntryTone**: `boolean`

In case of conference in this bubble, a local tone should be played when user joins. `Default : TRUE`

---

### noHistory?

> `optional` **noHistory**: `boolean`

If TRUE, the old history will NOT be shown to new users `Default : FALSE`

---

### privilege?

> `optional` **privilege**: [`BubbleUserPrivilege`](doc/sdk/webv2/lts/api/Bubbles/enumerations/BubbleUserPrivilege)

Add optional body parameter privilege ('user', 'moderator'). By default, all users get the simple 'user' privilege. Using 'privilege' parameter all invited users may have the same privilege in this bulk.

---

### topic?

> `optional` **topic**: `string`

The bubble description
