# Documentation

Telephony service

LTS release 4.0.0Published on 2022-12-21

## Telephony

This services manages PBX phone calls in a conversation. so it manages PBX calls between your PABX associated phone and a recipient's phone. If you don't have this service activated for your Rainbow user, all these methods will return an error when called. The main methods and events proposed in that service allow to:  
\- Know if this service is activated or not for the connected user,  
\- Know the version of the agent (deployed on the PBX) that monitors your line,  
\- Handle the basic telephony services: Make a call, take a call, hold a call, retrieve a call and release a call,  
\- Listen to the call state change Depending the agent version deployed, some services can return an error (unavailable service) when called

**Emits**: `event:RAINBOW_ONTELEPHONYSTARTED`, `event:RAINBOW_ONTELEPHONYSTOPPED`, `event:RAINBOW_ONTELEPHONYCALLSTATECHANGED`, `event:RAINBOW_ONVOIPCALLSTATECHANGED`, `event:RAINBOW_ONTELEPHONYFORWARDSTATECHANGED`, `event:RAINBOW_ONCALLNOMADICEVENT`, `event:RAINBOW_ONVOIPSTARTED`, `event:RAINBOW_ONVOIPSTOPPED`, `event:RAINBOW_ONVOICEMESSAGEUPDATED`, `event:RAINBOW_TELEPHONYMUTE`, `event:RAINBOW_ONTELEPHONYMODEUPDATED`, `event:RAINBOW_ONFORWARDMODEUPDATED`
