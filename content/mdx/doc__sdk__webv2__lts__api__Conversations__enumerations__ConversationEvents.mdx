# Enumeration: ConversationEvents

Conversation public events

## Events

### ON\_MESSAGES\_UPDATED

> **ON\_MESSAGES\_UPDATED**: `"ON_MESSAGES_UPDATED"`

This RBEvent is send when something occurs on conversation messages

---

### ON\_MESSAGE\_ADDED

> **ON\_MESSAGE\_ADDED**: `"ON_MESSAGE_ADDED"`

This RBEvent is send when a new message is added in this conversation

---

### ON\_MESSAGE\_BOOKMARKED

> **ON\_MESSAGE\_BOOKMARKED**: `"ON_MESSAGE_BOOKMARKED"`

This RBEvent is send when a bookmark is set in this conversation

---

### ON\_MESSAGE\_CACHED

> **ON\_MESSAGE\_CACHED**: `"ON_MESSAGE_CACHED"`

This RBEvent is send when a message is add in the "cache message". This "cache message" is used to store messages which belongs to the conversation but not already loaded. For example an old replied message.

---

### ON\_MESSAGE\_REPLACED

> **ON\_MESSAGE\_REPLACED**: `"ON_MESSAGE_REPLACED"`

This RBEvent is send when the replaced/deleted message is the last one

---

### ON\_MISSED\_COUNTER\_UPDATED

> **ON\_MISSED\_COUNTER\_UPDATED**: `"ON_MISSED_COUNTER_UPDATED"`

This RBEvent is send when the missed message counter is updated

---

### ON\_STATUS\_CHANGED

> **ON\_STATUS\_CHANGED**: `"ON_STATUS_CHANGED"`

This RBEvent is send when the conversation status is changed This status represent the user typing activity.

-   ACTIVE
-   INACTIVE
-   COMPOSING
-   PAUSED

---

### RAINBOW\_CONVERSATION\_CALL\_UPDATE

> **RAINBOW\_CONVERSATION\_CALL\_UPDATE**: `"RAINBOW_CONVERSATION_CALL_UPDATE"`

This RBEvent is send when In some situations when the call is "remotely picked" by another client and we update the tel call area (??) In some situations to manage the mute / unmute / sharing buttons in tell call area; This event is to be DELETED and replaced with something else;
