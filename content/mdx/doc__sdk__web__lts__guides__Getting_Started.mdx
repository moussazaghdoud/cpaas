## Getting Started

---

Welcome to the Rainbow Web Software Development Kit!

The Rainbow Web SDK (Software Development Kit) is a pure AngularJS JavaScript library that works on all modern browsers from IE 11 to the latest version of Chrome, Firefox, Safari and Edge. Its powerful APIs enable you to create the best JavaScript applications that connect to the Rainbow Cloud Services.

Getting started with Rainbow Web SDK could not be easier. By adding a single line to your existing JavaScript project, you will have access to the public API of the Rainbow Cloud Services.

**Events**

Events are asynchronous messages or notifications that come from Rainbow when something related to your data happens.  
By listening to theses events, your application will be able to react and inform the user of these changes.

**Methods**

Methods are all the SDK functions your application can call to enable Rainbow services such as initiating a WebRTC call, sending a file, etc. In most of the cases, Methods are asynchronous and follow the JavaScript Promise pattern.

### OS and browsers support

---

The Rainbow Web SDK is based on jQuery and the AngularJS framework.

The Rainbow Web SDK works on:

-   Chrome stable channel and Chromium based browser (e.g. Opera)
    
-   Firefox ESR and stable channel
    
-   Microsoft Edge
    
-   Microsoft Internet Explorer 11
    
-   Safari
    

The Rainbow Web SDK is supported on Windows (starting from Windows 7) and MacOSX (Starting from 10.10).

The Rainbow Web SDK can be integrated into any Web application (i.e.: React, Vue), browser extension or desktop application that embeds a Webview (e.g. QT application, Electron, nw.js). Some integration examples can be found on our [Github](https://github.com/Rainbow-CPaaS).

### What is the Rainbow Web SDK?

---

Rainbow Web SDK is a **JavaScript library** you have to use in order to connect and access Alcatel-Lucent Enterprise Rainbow.

Once loaded and initialized, you have access to the library by using the variable `rainbowSDK`.

This library is divided in **services** that let you access tons of **methods** and connect your app to the Rainbow functionalities.

For example, the `rainbowSDK.webRTC` is the service dedicated to WebRTC (audio/video calls). It will give you access to all the WebRTC available features that you can use. In that service you can use the method `rainbowSDK.webRTC.callInVideo()` to make a video call with another Rainbow user. To help you discover all the available services, you can have a look at the documentation accessible on the left side of this page.

Additionally, each service offers a set of **events** that you can listen to be notified of asynchronous things (e.g. incoming WebRTC call, etc.) that could occur.

#### Methods

---

There are two kinds of methods:

-   Those that deal with the Rainbow platform: For these APIs, a promise based object is returned in order to allow you to continue the execution of your code once the result of this API is known (good or not).
    
-   Those that only do things on the client side: For these APIs, the result is returned synchronously.
    

When you do not call the API with the right parameters, you always get a JSON result containing an error code and an error label that is much more explicit (...)

```javascript
/* Example of error code object */
{
  code: SDK.ERRORBADREQUEST,   // -16
  label: "Parameter 'contact' can't be null or empty"
}
```

#### Events

---

Events are initiated from the library when something that comes from the server happens.

They are asynchronous. It is not mandatory to listen to all the events that come from the server. Choose the right ones you need!
