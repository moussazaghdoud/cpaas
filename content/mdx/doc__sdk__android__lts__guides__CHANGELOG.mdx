### SDK for Android 2.11.0 - December 2021

---

#### API changes

In order to unify the SDK methods, we migrate the listener to a new type: `RainbowListener<T,U>`.

-   `T` represents the type of the returned data in case of success
-   `U` represents the type of the returned data in case of error

When there is no returned data, this listener returns the `Unit` type

This listener expose two methods :

-   `onSuccess(data: T)` this callback is called in case of success of the called function. `T` is the specified type in `RainbowListener<T,U>`
-   `onError(error: RainbowError<U>)` this callback is called in case of failure of the called function. `U` is the specified type in `RainbowListener<T,U>`

`RainbowError<U>` is a new type returned in the failure callback, it contains 3 fields :

-   `errorMsg` an error message
-   `errorCode` an optional error code
-   `errorType` an optional object of type `U`

-   The tutorial around the [`MyProfile`](/docs/sdk/android) module has been added. It explains how to get the connected user (+ his personal information) and how to update them.
-   The method `getUserPasswordInCache()` in `MyProfile` module has been added to retrieve the last password saved (last login).

#### BUGS

-   The method `RainbowSdk.instance().connection().isConnected()` has been fixed because it sometimes returned wrong value.

### SDK for Android 1.34 - Decembre 2017

---

#### SDK

-   The Rainbow SDK for Android is now hosted on our own Maven repository. **That's why, from mid-January (when the version 1.35.0 will be released), all versions on Github might be deleted. The Rainbow SDK for Android will soon be only available on Maven. Please change the way you import the SDK and use Maven.** See step 2 of the [Getting started](/docs/sdk/android).
-   The feature around channels has been added and is available by `RainbowSdk.instance().channels()`. This includes several methods (more detailled in the dedicated tutorial: [Managing channels](/docs/sdk/android)). - Create, delete or update a channel - `createChannel(name, topic, listener)` - `createPrivateChannel(name, topic, listener)` - `deleteChannel(channel, listener)` - `updateChannelDescription(channel, description, listener)` - Add or remove contact from a channel - `addMembersToChannel(channel, contacts, listener)` - `addOwnersToChannel(channel, contacts, listener)` - `addPublishersToChannel(channel, contacts, listener)` - `removeAllUsersFromChannel(channel, listener)` - `removeUsersFromChannel(channel, contacts, listener)` - Subscribe or unsubscribe to a channel (public channel) - `subscribeToChannel(channel, listener)` - `unsubscribeFromChannel(channel, listener)` - Get your channels - `getAllChannels()` - `getAllOwnedChannels()` - `getAllSubscribedChannels()` - Search channels - `findChannelsByName(name, listener)` - `findChannelsByTopic(topic, listener)` - `getChannelById(id, listener)` - Publish a message and listen for new ones - `publishMessage(channel, message, title, url, listener)` - `registerListener(listener)` - `unregisterListener(listener)`

#### BUGS

-   No bug fixing

-   **Compatibility Break** The methods `searchByName` and `searchByJid` have been changed: the parameter `IContactSearchListener` is replaced by `IRainbowContactsSearchListener` and the result of `searchFinished` is replaced from `List<DirectoryContact>` to `List<IContact>`.
