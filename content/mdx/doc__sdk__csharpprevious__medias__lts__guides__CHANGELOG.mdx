## SDK.Medias for C#

### \[1.0.0\] - 2024-07-01

#### Fixed

-   Improves the way to get/calculate Fps.
-   Avoid to use several streams when grabbing remote videos (for CCTV for example)

#### Changed

-   Use `FFmpeg.AutoGen` package v7.0.0 (from v6.0.0.2)
-   Use `FFmpeg v7.0.1` shared libraries (from v6.0)
-   **.Net 6.0** as target plateform has been removed (obsolete). **.Net Std 2.0, 2.1** and **.Net 8.0** are still in targets platform.

---

### \[0.0.9-STS\] - 2024-03-19

#### Fixed

-   Allow to stream as video a JPG file (already possible with BMP, TIFF, PNG, WEBP (with one image), ...). GIF are also supported but not WEBP with animation due to FFmpeg limitation

#### Changed

-   Use **Microsoft.Extensions.Logging.Abstractions v8.0.0.0** (from v7.0.0.0)

---

#### Added

-   Add .Net 8.0 as target plateform (.Net Std 2.0, 2.1 and .Net 6.0 are still in targets platform) .Net 6.0 target platform will be removed in April.

---

## Fix

-   **MediaRTPInput**: Some internal values was not resetted correctly
    
-   **MediaFiltered**: When a new filter is set, it's necessary to dispose ALL previous objects to ensure to build correct frames
    

### \[0.0.3\] - STS - 2023-02-10

---

## Enhancements

It's now possible **to decode RTP packets** (Audio and/or Video) using **MediaRtpInput** object.

A valid SDP must be provided which describes codecs to use to decode them. It possible for the moment to manage only one Audio track and/or one Video Track.

If you have several tracks for a same media, it's possible to specify which one to use using an index.

### \[0.0.2\] - STS - 2023-01-20

---

## Enhancements

-   Major rework to avoid memory leak. Use of **IDisposable** interface in objects using unmanaged objects to ensure to dispose them.
    
-   Rework interfaces (IMedia, IMediaVideo, IMediaAudio, IMediaVideoOutput, IMediaAudioOutput) and inheritances in major objects: **MediaInput**, **MediaFiltered** and **MediaOutput**
    

## API In details

-   **IMedia** interface (now inherits from **IDisposable**)
    
    -   `Loop`: **added** To know if we loop automatically if the end of file is reached
        
    -   `Duration`: **added** Duration in seconds of the Media (if not a life stream)
        
    -   `LiveStream`: **added** To know the Media is a 'live steam' i.e. a remote stream for a live stream server, live camera or an IPTV for example
        
    -   `OnEndOfFile`: **added** Event raised when the Media as reached the end of the file.
        
    -   `Init`: **added** To init the media.
        
-   **IMediaVideo** interface:
    
    -   `Loop`: **removed**
        
    -   `Duration`: **removed**
        
    -   `LiveStream`: **removed**
        
    -   `Init`: **removed**
        
-   **IMediaVideoOutput** interface: **added**
    
    -   `GotVideoFrame`: **added** To add Video Frame to this Media Video Output
-   **MediaFiltered** now inherits from **IMediaVideo** and **IMediaAudio**
    
-   **SDL2AudioInput** implements **IMedia** interface
    
-   **SDL2AudioOutput** implements **IMedia** interface
    
-   **MediaFormat** static class: **remoced** data/methods are now in **Helper** class
