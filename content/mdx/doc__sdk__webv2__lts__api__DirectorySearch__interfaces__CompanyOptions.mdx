# Interface: CompanyOptions

List of parameters associated with the company filter option.

## Properties

### companyId

> **companyId**: `string`

The id of a company to refine the search to include or exclude users/contacts of this company. 'companyId' option must be associated with a scope type to be used.

---

### companyScope

> **companyScope**: [`CompanyScopeType`](doc/sdk/webv2/lts/api/DirectorySearch/enumerations/CompanyScopeType)

This information indicates how the company specified via its id will be managed in the search scope. 'companyScope'' option must be associated with a company id to be used. When 'companyScope' is set to LIMITED\_TO\_COMPANY, then the search will be restricted to this company. When 'companyScope' is set to COMPANY\_EXCLUDED, then the search will exclude users or contacts being in this company. This option is managed differently depending on the directory type, as described below:

-   RAINBOW\_USERS\_NETWORK: This option is not available.
-   RAINBOW\_USERS: Both LIMITED\_TO\_COMPANY and COMPANY\_EXCLUDED options are available.
-   RAINBOW\_BUSINESS\_CONTACTS\_DIRECTORY: Only LIMITED\_TO\_COMPANY option can be specified.
-   RAINBOW\_PERSONAL\_CONTACTS\_DIRECTORY: This option is not available.
-   MICROSOFT\_O365\_USERS: This option is not available.
-   RAINBOW\_PBX\_USERS: This option is not available.

If the option value is not compatible with the specified directory, the request will be rejected and an error logged.
