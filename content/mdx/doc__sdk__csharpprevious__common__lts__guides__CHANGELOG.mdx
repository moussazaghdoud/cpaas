#### Fixed

-   The avatar of a bubble based on the avatars of its members will no longer contain the initials of one or more of these members

#### Changed

-   **Breaking** `GetBitmapFromSvgResourceName()` and `GetBitmapFromSvgFile()`: No longer return a square image but respect the SVG layout. A **width** parameter replaces the old **size** parameter. The **height** is automatically calculated.

-   Update all labels for 22 languages
-   Upgrade package `SkiaSharp` to 2.88.7 from 2.88.6 (since a rollback of dependencies used in this package has been done in this new version)
-   Use `Svg.Skia` 1.0.0.13 instead of `SkiaSharp.Svg` 1.60.0 to allow to manage more type of SVG

#### Added

-   Include more than 500 SVG (before ~60 SVG)

## Enhancements

**It's now possible to log on different files if several instance of the SDK (Core) is used in sametime** (for example if you use several bots in the same process).

You need to set to the new parameter `loggerPrefix` a different value for each instance when `Rainbow.Application` object is created. You also need to use a different configuration file for your logger. See the online documentation for more details.

In consequence the way to create these services have changed: Avatars, Languages and ImageTools:

```csharp
// Create Rainbow.Application - here with a logger prefix
String prefix = "BOT01_"; // This can be null or empty if you don't need it
var application = new Rainbow.Application(loggerPrefix: prefix);

// -------------------------------------------------

// Before Avatars service was created like this:
var avatars = Rainbow.Common.Avatars.Instance;
avatars.SetApplication(application);

// Now we use this to create Avatars service:
var avatars = Rainbow.Common.Avatars.CreateInstance(application);

// -------------------------------------------------

// Before Languages service was created like this:
var languages = Rainbow.Common.Languages.Instance;

// Now we use this to create Languages service:
var languages = Rainbow.Common.Languages.CreateInstance(application);

// -------------------------------------------------

// Before ImageTools service was composed of only static methods. Now you need to create it before to use any methods
var imageTools = Rainbow.Common.ImageTools.CreateInstance(application);
```

## \[2.1.0\] - LTS - 2021-05-24

---

### Files service: (new)

This service permits to facilitate the download of files descriptor, files thumbnail and files.

When a file descriptor has been downloaded:

-   if authorized, a thumbnail is automatically downloaded too.
-   if authorized and orignal file size enough smaller, the original image file is automatically downloaded instead of its thumbnail.

Once downloaded, thumbnails are automatically resized based on the max height or max width specified.

Thumbnails and originals files are stored in the specified folder.

### Avatars service:

-   Bubble avatar created with theirs members now use first creator, then moderators then members. Avoid current user if possible
-   Enhance download management on network trouble or on incorrect server responses

### ImageTools library:

-   Add 14 new Svg as resources in this library

### API update in details:

***Avatars service***: **updated**

-   `AddUnknownContactToPoolById()` **new** - To add contact avatar (by Id) to the download pool
-   `AddUnknownContactToPoolByJid()` **new** - To add contact avatar (by Jid) to the download pool
-   `GetTheoricContactAvatarPath` **new** - Returns the path of the contact Avatar according it's type only whatever the status of this avatar download.
-   `GetTheoricBubbleAvatarPath` **updated** - Returns the path of the bubble Avatar according it's type only whatever the status of this avatar download.
-   `BubbleAvatarUpdated` **updated** - This event is now raised only when all avatars members, used to create bubble avatar, are all downloaded.

***Files service***: **new**

*Events*

-   `FileDescriptorAvailable` **new** - Event raised when a FileDescriptor is available
-   `ThumbnailAvailable` **new** - Event raised when a thumbnail has been downloaded successfully and is available

*Properties*

-   `AllowAutomaticThumbnailDownload` **new** - To allow / disallow automatic thumbnail download - False by default
-   `AllowFileDescriptorDownload` **new** - To allow / disallow file descriptor download - True by default
-   `AutomaticDownloadLimitSizeForImages` **new** - To download original image and not thumbnail if its size is smaller than this property (in bytes) - 250 by default.
-   `MaxThumbnailHeight` **new** - Thumbnail will be resized automatically if its height is greater than this property - 200 by default
-   `MaxThumbnailWidth` **new** - Thumbnail will be resized automatically if its width is greater than this property - 200 by default

*Methods*

-   `AskFileDescriptorDownload(conversationId, fileDescriptorId)` **new** - To ask the download of a file descriptor in a conversation.
-   `AskThumbnailDownload(fileDescriptorId)` **new** - To ask the download of a thumbnail. The file descriptor must be available first.
-   `GetAllConversationIdByFilesDescriptorIdFromCache()` **new** - Get a dictionary of conversation id indexed by file descriptor id stored in the cache
-   `GetAllFilesDescriptorFromCache()` **new** - Get FileDescriptor object using fileDescriptorId.
-   `GetConversationIdByFileDescriptorId(fileDescriptorId)` **new** - To get the conversation Id related to the specified file descriptor Id.
-   `GetFileDescriptor(fileDescriptorId)` **new** - Get FileDescriptor object using fileDescriptorId.
-   `GetFullFilePath(fileDescriptorId)` **new** - Get in a dictionary (by file descriptor id) all files descriptor stored in cache
-   `GetThumbnailFullFilePath(fileDescriptorId)` **new** - Get file path to the thumbnail using file descriptor Id.
-   `IsThumbnailFileAvailable(conversationId, fileDescriptorId, fileName)` **new** - To know if thumbnail file is available
-   `SetApplication(application)` **new** - To set Rainbow Application object
-   `SetDensity(density)` **new** - Set the density used by the current device in order to resize correctly thumbnail - 1 by default
-   `SetFolderPath(folderPath)` **new** - Set folder path where thumbnails and files will be stored

***ImageTools library***: **updated**

-   `GetImageSize()` **new** - To get image size (using file path, SKImage or SKBitmap)
