CallLog Service

LTS release 2.XPublished on 2025-01-01

## CallLogService

This service allow to get the call log and manage it.  
  
The main methods and events proposed in that service allow to:  
\- Get all calls log  
\- Delete one or all calls log  
\- Mark calls as read / unread  

**Access**: public

### callLogService.isInitialized() ⇒ `Boolean`

Check if the call log history has been received from Rainbow  
A false answer means that the call logs have not yet been retrieved from the server.  

**Kind**: instance method of [`CallLogService`](#module_CallLogService)

**Returns**: `Boolean` - True if the call logs have been retrieved. False elsewhere.

**Category**: CallLog INITIALISATION

**Access**: public

**Nodered**: true

---

### callLogService.getAll() ⇒ `Array.<CallLog>`

**Kind**: instance method of [`CallLogService`](#module_CallLogService)

**Returns**: `Array.<CallLog>` - An array of call log entry

**Category**: CallLog MANAGEMENT

**Access**: public

### callLogService.getMissedCallLogCounter() ⇒ `Number`

Get the number of call missed (state === "missed" && direction === "incoming")  

**Kind**: instance method of [`CallLogService`](#module_CallLogService)
