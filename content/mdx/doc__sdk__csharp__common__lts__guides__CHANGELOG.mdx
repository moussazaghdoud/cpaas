### \[3.0.0-sts.3\] - 2024-10-01

#### Fix

-   `Avatars` service: Avoid to download / create some Avatars already available on restart
-   `Avatars` service: Event `PeerAvatarUpdated` is triggered as soon as possible - use `AllowAvatarDownload()` if you want to delay the process

#### Added

-   `Avatars` service: Manage also Avatars of Channels
-   `Avatars` service: `SetAvatarCategories()` to select avatar categories to manage (contact and bubble are used by default - not channel)

#### Changed

-   **Breaking** `Avatars` service: `CreateInstance()` has been renamed to `GetOrCreateInstance()`.
-   **Breaking** `Avatars` service: Remove `AddInDownloadPoolAllAvatarsFromContactsStoredInCache()` and `AddInDownloadPoolAllAvatarsFromBubblesStoredInCache()`. Use instead `SetAvatarCategory()`.
-   **Breaking** `Avatars` service: Remove `AddUnknownContactToPoolById()`, `AddUnknownContactToPoolByJid()` and `AllowToAskInfoForUnknownContact()` - when an avatar file path is asked for an unknown contact, he is automatically added to the download avatar pool.
-   **Breaking** `Avatars` service: Remove `AllowToAskInfoAboutUnknowBubble()`\- when an avatar file path is asked for an unknown bubble, he is automatically added to the download avatar pool.
-   `Avatars` service: Up to 3 avatars of members are used to create rounded bubble avatar (before it was up to 4)
-   `Avatars` service: The avatar of the bubble owner is taking now 50% of the space if 3 members are used to create the avatar
