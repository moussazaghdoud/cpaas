Device Management service (audio/video)

LTS release 4.0.0Published on 2022-12-21

## DeviceManagementService

This service manages the input / output audio and video devices used by the application

**Emits**: `event:RAINBOW_ONAUDIOVIDEODEVICEACCESSERROR`, `event:RAINBOW_ONAUDIOVIDEODEVICELISTUPDATE`, `event:RAINBOW_ONNEWAUDIOVIDEODEVICESELECTED`, `event:RAINBOW_ONNEWSECONDARYOUTPUTDEVICESELECTED`  
**Access**: public

### deviceManagementService.subscribe(handler) ⇒ `Subscription`

Subscribe to updates on the input/output devices (all events are of RBEvent type {name: string, data:any}); All public events that are fired by this service are described in the documentation. NB There might be some events that are not documented here, they should be ignored

**Kind**: instance method of [`DeviceManagementService`](#module_DeviceManagementService)  
**Returns**: `Subscription` - Returns RxJS Subscription  
**Access**: public

Param

Type

Description

handler

`any`

The call-back function that will be subscribed to the RxJS subject

### deviceManagementService.setSetSinkIdForAudioElement(element, outputDevice)

Set the output sinkID of the audio element with the output device given

**Kind**: instance method of [`DeviceManagementService`](#module_DeviceManagementService)  
**Access**: public

Param

Type

Description

element

`HTMLAudioElement`

The Audio element to whom we want to change the output ID

outputDevice

`MediaDeviceInfo`

The output device to be used

### deviceManagementService.getCurrentAudioInputDevice() ⇒ `MediaDeviceInfo`

**Kind**: instance method of [`DeviceManagementService`](#module_DeviceManagementService)  
**Returns**: `MediaDeviceInfo` - The current microphone MediaDeviceInfo or null if none

Param
