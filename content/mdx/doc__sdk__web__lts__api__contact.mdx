## Contact

This class is used to represent a contact or the connected user. A contact is defined by a set of public information (name, firstname, avatar) and a set of private information that are only shared with contacts that are in the user's network or in the same company (email, phone numbers)

### Contact~calendarInformation : `object`

EXPERIMENTAL (this property will likely change in the future version) Object containing all the calendar informations related to user The object contains the current meeting / incoming calendar meetings, but also it contains the Out Of Office informations: The structure of the object (by default it's empty if no calendar presence) : calendarInformation = { status: string //calendar status (one of "online", "dnd") mdate: date //the date of the end / next meeting autoReplyOn: boolean //if true, the user has activated Out Of Office) autoReplyInfos: { //if autoReplyOn is true, then this object contains the informations related to the Out Of Office) message: string //out of office message mdate: date //until when the OoO is activated (if known) } }

NB: The Out of Office is not automatic, so it's updated every 15min for each conversation that the user has;

**Kind**: inner property of [`Contact`](#module_Contact)  
**Access**: public  
**Read only**: true
